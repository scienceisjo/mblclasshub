목차

이 허브란 왜 좋은가 준비물 나만의 페이지 만들기 Supabase DB 구축 막히면

A이 허브란

MBL 센서 탐구 수업을 한 곳에 모으는 학생용 진입 페이지입니다.

학생은 학급코드·번호·PIN으로 로그인해 오늘의 활동(시뮬레이션·센서 탐구·영상·퀴즈)에 접근하고, 측정한 데이터와 형성평가 점수는 자동으로 기록됩니다. 선생님은 활동을 카드로 추가하고, 모인 데이터를 대시보드로 확인합니다. 화면 구성은 정적 웹페이지(GitHub Pages)가 담당하고, 로그인·저장·집계는 Supabase가 맡습니다.

B왜 좋은가

흩어진 자료와 활동을 하나로 묶고, 관리 부담은 낮춥니다.

한 곳에 모으기

흩어진 앱·자료·평가를 단일 진입점으로 묶습니다.

데이터 자동 수집

탐구 결과와 형성평가가 자동 저장돼 대시보드로 분석됩니다.

무료·무서버

GitHub Pages와 Supabase 무료 티어로 운영합니다.

관리가 쉬움

활동은 코드가 아니라 DB 행으로 추가돼 페이지가 길어지지 않습니다.

내 수업에 맞춤

제목·배경·자료·활동을 자유롭게 바꿉니다.

확장 가능

반별 과제, 파일 업로드, 데이터 분석 도구까지 이어붙일 수 있습니다.

C준비물

모두 무료이고, 계정 두 개면 시작할 수 있습니다.

1. GitHub 계정 — 페이지를 올리고 공개할 곳입니다.
2. Supabase 계정 — 로그인·데이터 저장을 맡는 DB입니다.
3. 파일 세트 — index.html, hub.js, schema.sql (이미 받으신 파일).
4. (선택) 이미지 — 배경·수업 사진.

D나만의 페이지 만들기

파일을 올리고, 이름·배경·활동만 내 수업으로 바꾸면 됩니다.

  1. 저장소 준비

    GitHub에서 새 저장소를 만듭니다. 개인 페이지로 쓰려면 이름을 아이디.github.io 로 만드세요. 그런 다음 index.html·hub.js·assets/ 를 올립니다. 처음이라면 저장소의 Add file → Upload files 로 드래그해 올리는 게 가장 쉽습니다.

  2. 제목·이름 바꾸기

    index.html 상단의 <title> 과 상단바 <div class="brand"> 안의 이름을 내 수업 이름으로 바꿉니다. 푸터의 이름도 함께 바꾸면 깔끔합니다.

  3. 배경·사진 넣기

    배경 이미지를 assets/lab-bg.jpg 로, 수업 사진을 assets/photos/ 에 올립니다. 배경 경로는 CSS 맨 위 --bg-image 한 줄에서 바꿀 수 있어요. 사진이 없어도 페이지는 어두운 배경으로 정상 표시됩니다.

  4. 자료 연결

    센서 가이드·수업 규칙 같은 자료는 guides/ 폴더에 PDF나 HTML로 올리고, 자료실 카드의 href 를 그 파일로 연결합니다(해당 카드 위 주석 참고).

  5. 활동 등록

    각 활동 앱(단일 HTML)을 activities/ 폴더나 별도 저장소에 배포한 뒤, Supabase activities 테이블에 행을 추가합니다(제목·유형·단원·차시·URL). 활동을 늘려도 index.html 은 그대로예요 — 페이지가 테이블을 읽어 카드를 그립니다.

  6. 배포

    저장소 Settings → Pages 에서 배포를 켭니다. 잠시 뒤 아이디.github.io 주소로 페이지가 공개됩니다.

큰 사진·영상·데이터셋은 저장소에 다 넣지 마세요. 파일 하나는 100MB, 저장소는 1GB 이하가 권장이라, 용량이 크면 Supabase Storage나 외부 링크로 빼는 게 좋습니다.

ESupabase DB 구축

로그인과 데이터 저장을 맡는 부분입니다. 순서대로 하면 15분이면 됩니다.

  1. 가입 · 프로젝트 생성

    supabase.com 에 가입해 New Project 를 만듭니다. 지역(Region)은 가까운 곳을 고르고, 데이터베이스 비밀번호는 잘 보관하세요.

  2. 스키마 실행

    왼쪽 SQL Editor 를 열고 schema.sql 전체를 붙여넣은 뒤 Run 합니다. 테이블·보안 정책(RLS)·로그인 함수가 한 번에 생성됩니다.

  3. 키 연결

    Settings → API 에서 Project URL 과 anon key 를 복사해 hub.js 맨 위 두 줄에 넣습니다.

    const SUPABASE_URL      = 'https://xxxx.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGciOi...';
    anon key는 화면에 노출돼도 안전합니다. 모든 접근이 RLS 정책으로 보호되기 때문입니다.

  4. 교사 계정 만들기

    Authentication → Users 에서 내 이메일로 계정을 하나 만듭니다. 이 계정이 학급·활동의 소유자(teacher_id)이자 대시보드 로그인 계정이 됩니다.

  5. 데이터 등록

    Table Editor 에서 세 가지를 입력합니다. classes 에 학급(학급코드 지정), students 에 학생(번호·닉네임·PIN), activities 에 활동(제목·유형·단원·차시·URL). 이때 teacher_id 는 방금 만든 내 계정의 UID를 넣습니다.

  6. 동작 확인

    허브 페이지에서 학급코드·번호·PIN으로 로그인해 봅니다. 활동 카드가 뜨면 성공입니다. 안 되면 키 입력, RLS 실행 여부, PIN을 순서대로 확인하세요.

PIN은 학급 안에서 겹치지 않게 배정하세요. 번호와 PIN을 함께 쓰기 때문에 도용이 어렵지만, 형성평가용 저부담 인증이라는 점을 전제로 사용하는 게 맞습니다.

F막히면

대부분의 문제는 세 가지 중 하나입니다. 키가 안 맞거나(hub.js의 URL·anon key), 스키마를 안 돌렸거나(SQL Editor에서 Run), PIN이 다른 경우예요. 순서대로 확인하면 거의 해결됩니다. 반별 과제·파일 업로드·데이터 분석 같은 확장은 스키마를 한 겹 더 얹으면 되니, 기본이 도는 걸 먼저 확인하고 진행하세요.
