목차

이 허브란 왜 좋은가 준비물 나만의 페이지 만들기 Supabase DB 구축 막히면

A이 허브란

MBL 센서 탐구 수업을 한 곳에 모으는 학생용 진입 페이지입니다.

학생은 학급코드·번호·PIN으로 로그인해 오늘의 활동(시뮬레이션·센서 탐구·영상·퀴즈)에 접근하고, 측정한 데이터와 형성평가 점수는 자동으로 기록됩니다. 화면은 상단 탭으로 전환돼요 — 탐구 흐름·오늘의 수업·활동·자료실·안내로 나뉘어, 길게 스크롤하지 않고 필요한 곳만 봅니다. 페이지 구성은 GitHub Pages가, 로그인·저장·집계는 Supabase가 맡습니다.

B왜 좋은가

흩어진 자료와 활동을 하나로 묶고, 관리 부담은 낮춥니다.

한 곳에 모으기

흩어진 앱·자료·평가를 단일 진입점으로 묶습니다.

데이터 자동 수집

탐구 결과·형성평가가 자동 저장돼 대시보드로 분석됩니다.

탭으로 깔끔하게

스크롤 대신 탭 전환이라 학생이 필요한 화면만 봅니다.

관리가 쉬움

활동은 코드가 아니라 DB 행으로 추가돼 페이지가 길어지지 않습니다.

무료·무서버

GitHub Pages와 Supabase 무료 티어로 운영합니다.

확장 가능

반별 과제, 파일 업로드, 데이터 분석까지 이어붙일 수 있습니다.

C준비물

모두 무료이고, 계정 두 개면 시작할 수 있습니다.

1. GitHub 계정 — 페이지를 올리고 공개할 곳입니다.
2. Supabase 계정 — 로그인·데이터 저장을 맡는 DB입니다. 기존에 쓰던 프로젝트가 있어도, 이 허브의 테이블은 모두 hub_ 로 시작해 겹치지 않습니다.
3. 파일 세트 — index.html, hub.js, manual.html, schema.sql.
4. 이미지 — 배경이 아니라 섹션 사진 10장(선택, 없어도 자리 안내가 표시됨).

D나만의 페이지 만들기

파일을 올리고, 이름·이미지·활동만 내 수업으로 바꾸면 됩니다.

  1. 저장소 준비

    GitHub 저장소에 index.html·hub.js·manual.html·assets/ 를 올립니다. index.html 과 hub.js 는 반드시 같은 폴더에 나란히 두세요(index 가 ./hub.js 로 불러옵니다). 처음이면 Add file → Upload files 로 드래그해 올리는 게 가장 쉽습니다.

  2. 제목·이름 바꾸기

    index.html 의 <title>, 상단바 <div class="brand">, 푸터의 이름을 내 수업 이름으로 바꿉니다. 문의처(이메일·인스타)도 푸터 근처에서 수정하면 됩니다.

  3. 이미지 넣기

    사진은 assets/ 폴더 하나에 아래 이름 그대로 넣습니다(확장자 .png). 없는 사진은 "○○ 이미지 자리" 안내가 뜨고 레이아웃은 그대로 유지됩니다.

    파일명쓰이는 곳
    hero.png메인 히어로 대표 이미지
    sensors.png센서 알아보기 (흐름)
    activity.png탐구 활동 (흐름)
    analysis.png데이터 분석 (흐름)
    portfolio.png결과 정리·공유 (흐름)
    mission.png오늘의 미션 배너
    my-record.png나의 기록 (로그인 후)
    safety.png안전 수칙
    help.png도움말
    class-01.png자료실 수업 사진

  4. 자료 연결

    센서 가이드·수업 규칙 같은 자료는 저장소에 파일로 올리고, 자료실 카드의 href 를 그 파일로 연결합니다(해당 카드 위 주석 참고). 연결하면 "준비 중" 표시를 "열기"로 바꿔 주세요.

  5. 활동 등록

    각 활동 앱(단일 HTML)을 배포한 뒤, Supabase hub_activities 테이블에 행을 추가합니다 — 제목·유형(simulation/inquiry/video/quiz)·학년(grade)·단원(unit)·차시(lesson_no)·URL. 활동을 늘려도 index.html 은 그대로예요. 카드의 "중1 - 빛과 파동 - 2차시" 표시가 이 값들에서 자동으로 채워집니다.

  6. 배포

    저장소 Settings → Pages 에서 배포를 켭니다. 잠시 뒤 아이디.github.io/저장소이름/ 주소로 공개됩니다(예: scienceisjo.github.io/mblclasshub/).

큰 사진·영상·데이터셋은 저장소에 다 넣지 마세요. 파일 하나 100MB, 저장소 1GB 이하가 권장이라, 용량이 크면 Supabase Storage 나 외부 링크로 빼는 게 좋습니다.

ESupabase DB 구축

로그인과 데이터 저장을 맡는 부분입니다. 순서대로 하면 15분이면 됩니다.

  1. 가입 · 프로젝트 생성

    supabase.com 에 가입해 New Project 를 만듭니다. 지역은 가까운 곳을 고르고, DB 비밀번호는 잘 보관하세요.

  2. 스키마 실행

    왼쪽 SQL Editor 에 schema.sql 전체를 붙여넣고 Run 합니다. hub_classes·hub_students·hub_activities·hub_submissions·hub_scores 다섯 테이블과 보안 정책(RLS)·로그인 함수가 한 번에 생성됩니다. 모두 hub_ 로 시작해 기존 테이블과 충돌하지 않습니다.

  3. 키 연결

    Settings → API 에서 Project URL 과 anon key 를 복사해 hub.js 맨 위 두 줄에 넣습니다.

    const SUPABASE_URL      = 'https://xxxx.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGciOi...';
    anon key 는 공개돼도 안전합니다(RLS 보호). 반면 service_role key 는 절대 코드·저장소에 넣지 마세요 — RLS 를 무시하는 관리자 키입니다.

  4. 교사 계정 만들기

    Authentication → Users 에서 내 이메일로 계정을 만듭니다. 이 계정이 학급·활동의 소유자(teacher_id)이자 대시보드 로그인 계정이 됩니다.

  5. 데이터 등록

    Table Editor 에서 hub_classes 에 학급(학급코드 지정), hub_students 에 학생(번호·닉네임·PIN), hub_activities 에 활동을 입력합니다. teacher_id 는 내 계정의 UID 를 넣으세요.

  6. 동작 확인

    허브에서 학급코드·번호·PIN 으로 로그인해 봅니다. 활동 카드가 뜨면 성공입니다.

PIN 은 학급 안에서 겹치지 않게 배정하세요. 번호와 PIN 을 함께 쓰기 때문에 도용이 어렵지만, 형성평가용 저부담 인증이라는 전제로 사용하는 게 맞습니다.

F막히면

대부분의 문제는 셋 중 하나예요. 키가 안 맞거나(hub.js 의 URL·anon key), 스키마를 안 돌렸거나(SQL Editor 에서 Run), PIN 이 다른 경우입니다. 순서대로 확인하면 거의 해결됩니다. 반별 과제·파일 업로드·데이터 분석 같은 확장은 스키마를 한 겹 더 얹으면 되니, 기본이 도는 걸 먼저 확인하고 진행하세요.
